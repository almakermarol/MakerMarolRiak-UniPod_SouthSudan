SSH Key Setup

Understanding SSH Authentication

SSH keys provide a secure, password-less way to connect your computer to GitHub. The analogy is creating a unique matched lock (public key) and key (private key) - you give GitHub the lock, and you keep the key.

SSH Key Generation Process

We generated ED25519 SSH key pairs using the terminal. When prompted during generation, we accepted the default file location and could choose to create a passphrase for additional security or proceed without one for simplicity.

Adding Public Key to GitHub

  1. Displayed and copied the public key from the terminal
  2. Navigated to GitHub.com → Settings → SSH and GPG keys → New SSH Key
  3. Added a descriptive title (e.g., "My Laptop") and pasted the public key
  4. Clicked "Add SSH Key" to complete the setup

Repository Management

Creating Your First Repository

  1. Clicked the "+" icon and selected "New repository"
  2. Set repository name using the format: yourname-UniPod_Name
  3. Added description: "My portfolio for the UniPod Digital Fabrication Program."
  4. Set visibility to Public
  5. Checked "Add a README file" option
  6. Clicked "Create repository" to finalize

Forking Existing Repositories

Forking allows you to copy a repository from one account to another, enabling experimentation without affecting the original project.

  1. Navigated to the repository we wanted to fork
  2. Clicked the "Fork" button in the top-right corner
  3. Selected our personal account as the destination
  4. Renamed the forked repository in the Settings tab

Cloning to Local Machine

We cloned our repositories to create local folders on our computers that are linked to remote GitHub repositories. This synchronized local copy allows us to work offline and push changes seamlessly back to the cloud.

Git Workflow

The Three-Step Dance

Git uses a precise three-step process for managing changes:

  • git add: Stage your changes (preparing items for shipping)
  • git commit: Seal the package with a descriptive message (saving changes locally)
  • git push: Ship to GitHub (uploading changes to remote repository)

Making and Committing Changes

After cloning our repository, we made our first edit in VS Code by adding a welcome message to the README.md file. We then applied the Git workflow to save these changes and synchronize them with our GitHub repository.

GitHub Pages Deployment

Turning Your Repository into a Website

  1. Went to our repository on GitHub
  2. Clicked the "Settings" tab
  3. In the left sidebar, clicked "Pages"
  4. Under "Source," selected "Deploy from a branch"
  5. Under "Branch," selected the main branch and the / (root) folder
  6. Clicked "Save" to deploy the site

Our sites became live at URLs following this pattern: https://your-username.github.io/unipod-portfolio/

Note: It can take a few minutes for the site to become available for the first time.

Reflection

Learning Outcomes

This week established the foundation for professional documentation practices that will support our entire learning journey. Mastering Git and GitHub is essential not just for this program but for modern technical careers. The SSH key setup, while initially challenging, provides a more secure and efficient authentication method than passwords.

Technical Challenges

  • Understanding the distinction between Git (local version control) and GitHub (remote hosting platform)
  • Proper SSH key generation, especially file permissions and directory setup
  • Mastering the Git workflow: understanding when to use add, commit, and push
  • Configuring GitHub Pages correctly for the first deployment
  • Learning to write effective commit messages that clearly describe changes

Key Takeaways

  • Documentation is indeed a superpower for remembering, collaborating, and career development
  • The Git workflow (add → commit → push) becomes intuitive with practice
  • SSH keys, once properly set up, create a seamless authentication experience
  • GitHub Pages provides an incredibly simple way to deploy static websites
  • Professional username and repository naming matters for portfolio presentation
